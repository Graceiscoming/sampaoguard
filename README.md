# แพลตฟอร์มระบบธนาคารจำลอง Sampaoguard (ความปลอดภัยทางการเงินแบบ End-to-End)

โปรเจกต์นี้ได้รับการพัฒนาขึ้นเพื่อนำเสนอวิสัยทัศน์ทางวิศวกรรมซอฟต์แวร์ในการแก้ไขช่องโหว่การโอนเงิน (ภัยไซเบอร์, มิจฉาชีพสวมรอย QR, การฉ้อโกงสลิป และคดีขู่กรรโชกทรัพย์) โดยประมวลผล **AI แบบ Local 100% ฟรี ไม่มีค่าใช้จ่าย** และรักษาความเป็นส่วนตัวของข้อมูลผู้ใช้อย่างสูงสุด

---

## 🚀 8 ฟีเจอร์หลักแบ่งตามเลเยอร์ความปลอดภัย

### 🟡 Layer 1: ตรวจสอบและป้องกันก่อนโอน (Pre-Transaction)
1. **QR-Overlay Shield:** ใช้ OpenCV ตรวจสอบขอบป้ายสติกเกอร์ที่อาจนำมาแปะทับป้ายจริง (ตรวจพบขอบซ้อนเหลื่อมกัน)
2. **Liveness & Screen Glare Detection:** ตรวจหาแสงสะท้อนจ้า (Specular Reflection) ผิดปกติ เพื่อจำแนกว่าเป็นการสแกนจากหน้าจอ iPad/มือถือเครื่องอื่น (หลอกสแกน) ไม่ใช่ป้ายกระดาษจริง
3. **GPS Distance Match:** ครอสเช็กพิกัดผู้ใช้ ณ ขณะสแกนเทียบกับสถานที่ตั้งร้านค้าจริง หากห่างกันเกิน 50 เมตร ระบบจะขึ้นแจ้งเตือนทันที

### 🟡 Layer 2: ลดความเสี่ยงและเอาตัวรอดระหว่างโอน (During-Transaction)
4. **Auto Time-Locked Escrow:** เมื่อโอนไปยังบัญชีบุคคลทั่วไปที่ไม่มีประวัติธุรกรรม ระบบจะหน่วงเงินไว้ในเซิร์ฟเวอร์ 60 วินาที (ในการเดโม่) และแสดงปุ่ม **"Recall" (ดึงเงินคืน)** เพื่อดึงเงินกลับเข้าบัญชีทันทีหากรู้ตัวว่าโดนหลอก
5. **Smart Trust-Based Routing:** วิเคราะห์คะแนนความน่าเชื่อถือปลายทาง (Trust Score) เช่น นิติบุคคล หรือร้านค้ายืนยันตัวตน เงินจะถูกส่งทันทีโดยข้ามขั้นตอนการหน่วงเงิน (Escrow)
6. **Duress PIN / Ghost Transfer (รหัสผ่านภัยพิบัติ):** เมื่อผู้ใช้ถูกโจรจี้บังคับโอนเงิน ให้ป้อนรหัส PIN ฉุกเฉิน **`999999`** ระบบจะตัดยอดโอนเงินจริงทิ้ง แต่หน้าจอยังแสดงยอดและออกสลิปจำลองว่าโอนสำเร็จเพื่อตบตาโจร และแอบส่งพิกัดแจ้งเตือนเงียบไปยังศูนย์ตำรวจ (Silent Alarm Webhook)
7. **Decoy Vault (บัญชีหน้าม้าลวงตา):** หน้าบัญชีธนาคารจะสลับเป็นบัญชีปลอมแสดงยอดเงินน้อย (เช่น ฿1,200) ทันทีที่ป้อนรหัสภัยพิบัติ ป้องกันไม่ให้โจรเห็นยอดเงินจริง

### 🟢 Layer 3: ยืนยันความโปร่งใสหลังโอน (Post-Transaction)
8. **Edge-AI Fake Slip Scanner:** วิเคราะห์รูปสลิปด้วยเทคโนโลยี **Error Level Analysis (ELA)** เพื่อดูการแต่งภาพบริเวณตัวเลขยอดเงิน และรัน **EasyOCR** ดึงตัวอักษรเพื่อวิเคราะห์ความขัดแย้งเชิงบริบท
9. **Micro-Cryptographic Watermark:** ฝังลายน้ำดิจิทัลเข้ารหัสลับลายเซ็น (`HMAC`) ลงในระดับพิกเซลรูปภาพสลิปที่ถูกสร้างขึ้น เมื่อร้านค้าสแกนสลิป ระบบจะแกะรหัสออกมาตรวจสอบ หากไม่มีหรือลายเซ็นไม่ตรง จะระบุว่าสลิปผ่านการดัดแปลงทันที

## 🛠️ โครงสร้างเทคโนโลยี (Tech Stack)

- **Frontend:** Next.js 14/16 (App Router) + Tailwind CSS + Lucide Icons + Framer Motion (สำหรับแอนิเมชันตอนโดนบีบบังคับสลับ Decoy Vault ให้ดูเนียนตา)
- **Backend:** Node.js Express.js + JSON Database (ความน่าเชื่อถือสูง ไม่ต้องลงฐานข้อมูลเพิ่ม)
- **AI-Service:** Python 3.10+ FastAPI + OpenCV (ประมวลผลความละเอียดภาพและ ELA) + **EasyOCR** (ทำ OCR ตัวอักษรไทย-อังกฤษแบบออฟไลน์)
- **Local LLM Integration:** เชื่อมต่อกับ **Ollama** บนเครื่องผู้ใช้งานโดยตรงเพื่อทำความเข้าใจเนื้อหาสลิป (เช่น ใช้โมดูล Qwen2 หรือ Llama3)

## ⚡ วิธีการติดตั้งและรันระบบ (Setup & Running)

### 1. ความต้องการของระบบ (Prerequisites)
- มี **Node.js** ติดตั้งในเครื่อง (เวอร์ชัน 18 ขึ้นไป)
- มี **Python** (เวอร์ชัน 3.10 ขึ้นไป)
- (ทางเลือก) ติดตั้ง **Ollama** และดาวน์โหลดโมเดลภาษาโดยใช้คำสั่ง:
  ```bash
  ollama run qwen2:1.5b
  ```
  *(หมายเหตุ: หากยังไม่มี Ollama ระบบแกะสแกนสลิปออฟไลน์ด้วย EasyOCR จะทำงานเป็นโหมดทางเลือกทดแทนให้อัตโนมัติโดยไม่เกิดข้อผิดพลาด)*

### 2. วิธีรันระบบทั้งหมดด้วยคลิกเดียว
เพื่อความสะดวกสำหรับงานนำเสนอและการนำเดโม่ไปรันในเครื่องกรรมการ คุณสามารถดับเบิ้ลคลิกไฟล์รันด่วนได้ทันที:
- รันไฟล์: **`run.bat`** (สคริปต์จะทำการเปิด Command Prompt 3 หน้าต่างขึ้นมารัน Frontend, Backend, และ AI-Service ให้พร้อมใช้งานทันที)

## 🚦 ลำดับการทดสอบระบบ (Demo Scenarios)

1. **โหมดโอนเงินปกติ (Normal Mode):**
   - เข้าสู่ระบบที่หน้าเว็บด้วยรหัส PIN: **`123456`**
   - หน้าจอจะแสดงบัญชีหลัก ยอดเงิน **฿450,000**
   - กดปุ่ม **"โอนเงิน / สแกน QR"** และเลือก **"จำลองโอนเงินนิติบุคคล"** หรือ **"จำลองโอนเงินร้านค้าตลาดสด"** -> โอนสำเร็จทันทีเพราะมี Trust Score สูง

2. **โหมดหน่วงเวลา Escrow และ Recall:**
   - กดปุ่ม **"โอนเงิน / สแกน QR"** อีกครั้ง เลือกปุ่ม **"⚠️ มิจฉาชีพแปะป้าย QR สวมรอย"**
   - ระบบจะตรวจพบกรอบป้ายสติกเกอร์ซ้อนทับกัน (QR-Overlay) แสงสะท้อนจ้า (Glare) และตำแหน่งสแกนคลาดเคลื่อน (GPS Mismatch) พร้อมระบุว่า Trust Score ต่ำมาก (20%)
   - ดำเนินการกดโอนเงิน -> ระบบจะบังคับหน่วงเวลาคุ้มครองภัย (Escrow Hold) 60 วินาที
   - กลับไปหน้าบัญชีหลัก คุณจะพบเวลาการโหนเงินนับถอยหลัง ให้ลองกดปุ่ม **"ดึงเงินคืนทันที (Recall)"** เพื่อทดสอบการดึงเงินกลับ ยอดเงินจะกลับมาทันที

3. **โหมดเผชิญหน้าโจร (Duress PIN & Decoy Vault):**
   - เข้าสู่ระบบด้วย PIN ภัยพิบัติ: **`999999`** (หรือกดยืนยันการโอนเงินหน้าสุดท้ายด้วย PIN นี้)
   - หน้าจอธนาคารจะสลับเป็นสีเหลืองอ่อน ยอดเงินเหลือเพียง **฿1,200** และรายการประวัติธุรกรรมปลอมจะถูกวาดแทนที่
   - ในขณะเดียวกัน ให้เปิดหน้าจอ **"สถานีตำรวจ"** (ลิงก์ที่ท้ายหน้าหลัก หรือหน้าเว็บบอร์ดตำรวจที่ http://localhost:3000/police) -> จะพบการแจ้งเหตุร้ายสีแดงและพิกัดแผนที่เรดาร์การแจ้งเตือนเงียบ (Silent Alarm) ปรากฏทันที

4. **โหมดแม่ค้ารับเงินสแกนสลิปปลอม (Merchant Scan):**
   - ไปที่เมนูหน้าร้านค้าสแกนสลิป (http://localhost:3000/merchant)
   - ดาวน์โหลดรูปสลิปสำเร็จที่สร้างมาหลังจากโอนเงินปกติ นำมาอัปโหลดตรวจสอบ -> ระบบจะบอกว่า **VALID** (ลายน้ำถูกต้อง พิกเซลภาพไม่มีรอยดัดแปลง)
   - หากลองแก้ไฟล์ภาพสลิปโดยวาดขีดเขียนทับยอดเงินด้วยโปรแกรม Paint/Photoshop แล้วอัปโหลด -> ระบบจะแสดงแผนผังความร้อนสีส้ม-ขาวขึ้นมาบนจุดที่แต่งรูป (ELA Heatmap) และแจ้งสถานะสลิปมีความเสี่ยงสูงเนื่องจากลายน้ำดิจิทัลถูกทำลาย
