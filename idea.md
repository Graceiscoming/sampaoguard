# แพลตฟอร์มเว็บแอปพลิเคชันเพื่อความปลอดภัยในการทำธุรกรรมการเงินแบบ End-to-End (Sampaoguard)

แพลตฟอร์มนี้ออกแบบมาเพื่ออุดช่องโหว่ทางกายภาพและจิตวิทยาที่ระบบธนาคารปัจจุบันยังครอบคลุมไม่ถึง โดยจำลองหน้าตา UI เป็นแอปพลิเคชันธนาคาร (ผ่าน Web App) 

โดยมีทั้งหมด 8 ฟีเจอร์หลัก แบ่งการทำงานออกเป็น 3 ระยะ ดังนี้:

## 🟡 Layer 1: ตรวจสอบและป้องกัน "ก่อนโอน" (Pre-Transaction)

1. **QR-Overlay Shield (ระบบตรวจจับสติ๊กเกอร์แปะทับป้าย QR)**
   - **การทำงาน:** ใช้เทคโนโลยี AI (Computer Vision) ประมวลผลภาพจากกล้องมือถือ เพื่อหาร่องรอยขอบสติ๊กเกอร์หรือความหนาที่ซ้อนทับกันบนป้ายร้านค้า ควบคู่กับการครอสเช็กพิกัดตำแหน่ง (Geolocation) ปัจจุบันกับสถานที่จดทะเบียนบัญชีนั้นๆ
   - **แก้ปัญหาอะไร:** แก้ปัญหามิจฉาชีพแอบนำสติ๊กเกอร์ QR Code บัญชีม้าของตัวเอง ไปแปะทับป้าย QR ของร้านค้าตามตลาดนัด หรือแปะทับ QR ตู้รับบริจาค

2. **Liveness & Screen Glare Detection (ระบบตรวจจับแสงหน้าจอและป้องกัน Fake GPS)**
   - **การทำงาน:** ทำงานเสริมฟีเจอร์ที่ 1 โดยวิเคราะห์ "แสงสะท้อน" (Glare) เพื่อแยกว่าสิ่งที่ผู้ใช้กำลังสแกนคือ "กระดาษป้ายจริงๆ" หรือ "หน้าจอมือถือ/แท็บเล็ต" พร้อมระบบบล็อกการใช้แอปปลอมแปลงตำแหน่งที่ตั้ง
   - **แก้ปัญหาอะไร:** อุดช่องโหว่กรณีที่มิจฉาชีพไม่ได้ใช้กระดาษแปะทับ แต่ใช้หน้าจอ iPad เปิดรูป QR ขึ้นมาหลอกให้สแกนโดยตรง รวมถึงกันมิจฉาชีพแอบอ้างว่าอยู่พื้นที่เดียวกับผู้ใช้

## 🟡 Layer 2: ลดความเสี่ยงและเอาตัวรอด "ระหว่างโอน" (During-Transaction)

3. **Auto Time-Locked Escrow (ระบบหน่วงเวลาการโอนเงิน)**
   - **การทำงาน:** เมื่อมีการโอนเงินแบบ P2P (บุคคลสู่บุคคล) ไปยัง "บัญชีแปลกหน้าที่ไม่เคยมีประวัติโอนหากันมาก่อน" ระบบจะ Hold เงินจำนวนนั้นไว้ 15-30 นาที (สถานะ Pending) หากผู้โอนรู้ตัวว่าโดนหลอก สามารถกดปุ่ม "ดึงเงินกลับ (Recall)" ในแอปได้ทันที
   - **แก้ปัญหาอะไร:** แก้ปัญหาการซื้อของออนไลน์ผ่านโซเชียลที่ "โอนปุ๊บ โดนบล็อกหนีปั๊บ" ช่วยให้ผู้ใช้ดึงเงินคืนได้ทันโดยไม่ต้องเสียเวลาไปแจ้งความหรืออายัดบัญชี

4. **Smart Trust-Based Routing (ระบบคัดกรองความน่าเชื่อถือบัญชี)**
   - **การทำงาน:** ทำงานเสริมฟีเจอร์ที่ 3 โดยนำ Trust Score มาใช้ หากเป็นการโอนเข้าบัญชีนิติบุคคล, ร้านค้าประจำ, หรือพร้อมเพย์เบอร์โทรศัพท์ที่ยืนยันตัวตนแน่นหนา เงินจะวิ่งเข้าปลายทางทันที (Instant Settlement) โดยไม่ถูกหน่วงเวลา
   - **แก้ปัญหาอะไร:** อุดช่องโหว่ไม่ให้ระบบหน่วงเงินไปสร้างความเดือดร้อนให้พ่อค้าแม่ค้าตัวจริงที่ต้องใช้เงินหมุนเวียน และป้องกันปัญหาคนซื้อรับของแล้วหัวหมอกดดึงเงินคืน (Buyer Fraud)

5. **Duress PIN / Ghost Transfer (ระบบรหัสผ่านฉุกเฉินและโหมดจำยอม)**
   - **การทำงาน:** ให้ผู้ใช้ตั้งรหัส PIN ที่สอง (เช่น ปกติ 1234 รหัสฉุกเฉิน 4321) หากกดรหัสฉุกเฉิน ระบบจะโชว์หน้าจอสร้างสลิปปลอมว่า "โอนเงินสำเร็จแล้ว" แต่ระบบหลังบ้านจะตัดการโอนเงินจริง และแอบส่งพิกัดแจ้งเตือนตำรวจ (Silent Alarm)
   - **แก้ปัญหาอะไร:** ช่วยชีวิตผู้ใช้ในกรณีถูกโจรจี้ปล้นบังคับให้โอนเงิน หรืออยู่ในสภาวะตื่นตระหนกขั้นสุดจากการถูกแก๊งคอลเซ็นเตอร์ขู่กรรโชกและกดดัน

6. **Decoy Vault (ระบบบัญชีหน้าม้าลวงตา)**
   - **การทำงาน:** ทำงานเสริมฟีเจอร์ที่ 5 ทันทีที่ผู้ใช้กดรหัส PIN ฉุกเฉิน หน้าแอปจะสลับหน้าตา UI ไปยัง "บัญชีจำลอง" ที่เซ็ตไว้ล่วงหน้า เช่น แสดงให้เห็นว่ามียอดเงินคงเหลือทั้งบัญชีแค่ 1,000 บาท
   - **แก้ปัญหาอะไร:** อุดช่องโหว่กรณีที่โจรขอดูหน้าจอแอปธนาคารเพื่อเช็กยอดเงินคงเหลือ ฟีเจอร์นี้จะหลอกโจรให้เชื่อสนิทใจว่าเราหมดตัวแล้วจริงๆ เพื่อให้โจรยอมปล่อยตัวและไม่ทำร้ายร่างกายต่อ

## 🟢 Layer 3: ยืนยันความโปร่งใส "หลังโอน" (Post-Transaction)

7. **Edge-AI Fake Slip Scanner (ระบบสแกนสลิปปลอมแบบออฟไลน์)**
   - **การทำงาน:** ฟีเจอร์ฝั่งร้านค้ารับเงิน โดยใช้ AI ขนาดเล็กที่ฝังอยู่ในแอปมือถือ (On-device AI) สแกนสลิปในมือถือของลูกค้า เพื่อตรวจจับร่องรอยการตัดต่อพิกเซล การเปลี่ยนฟอนต์ตัวเลข หรือแสงเงาที่ผิดปกติ โดยทำงานได้ทันทีไม่ต้องง้ออินเทอร์เน็ต
   - **แก้ปัญหาอะไร:** แก้ปัญหาลูกค้าใช้แอปแต่งรูปแก้ตัวเลขยอดเงินในสลิป แล้วเอามาโชว์หลอกพ่อค้าแม่ค้าในจังหวะที่คนหน้าร้านกำลังวุ่นวายและไม่มีเวลาเช็กแอปธนาคาร

8. **Micro-Cryptographic Watermark (ระบบลายน้ำดิจิทัลเข้ารหัสขั้นสูง)**
   - **การทำงาน:** กำหนดมาตรฐานใหม่ให้แอปธนาคารต้องฝัง "ลายน้ำเข้ารหัสลับที่มองไม่เห็น (Invisible Signature)" ลงไปในระดับพิกเซลของภาพสลิปที่ถูกสร้างขึ้น เมื่อ AI ของแม่ค้าสแกนสลิป จะทำการถอดรหัสนี้ด้วย หากไม่มีแปลว่าเป็นสลิปที่ไม่ได้มาจากระบบธนาคาร
   - **แก้ปัญหาอะไร:** อุดช่องโหว่กรณีมิจฉาชีพสาย Tech ที่ไม่ใช้แอปแต่งรูป (ทำให้ไม่มีรอยตัดต่อพิกเซล) แต่เขียนโปรแกรม Render ภาพสลิปขึ้นมาใหม่ทั้งใบ ซึ่งระบบลายน้ำนี้จะทำให้การทำสลิปปลอมด้วยวิธีนี้เป็นไปไม่ได้เลย 100%

## 💻 Tech Stack & Architecture (สถาปัตยกรรมระบบแบบละเอียด)

เพื่อให้ระบบทำงานได้ลื่นไหล รองรับฟีเจอร์ AI และมีความน่าเชื่อถือในระดับ Enterprise (เพื่อโชว์วิสัยทัศน์ทางวิศวกรรมซอฟต์แวร์) โครงสร้างระบบจะถูกแบ่งออกเป็น Microservices ย่อยๆ ดังนี้:

### 1. Frontend (Client-side Web Application)
*   **Framework:** **Next.js 14 (App Router)** ร่วมกับ React 18 เพื่อรองรับการทำ SSR (Server-Side Rendering) และเพิ่มความเร็วในการโหลด
*   **Styling & UI:** **Tailwind CSS** สำหรับสร้างดีไซน์ที่ดูพรีเมียม (Premium Banking UI) เช่น Glassmorphism และใช้ **Framer Motion** สำหรับแอนิเมชันตอนเปลี่ยนหน้าหรือสลับไป Decoy Vault ให้ดูเนียนตา
*   **PWA (Progressive Web App):** ใช้ `next-pwa` เพื่อให้ผู้ใช้สามารถกด "Add to Home Screen" ได้ ทำให้เว็บแอปทำงานแบบ Full-screen คล้าย Native App บนมือถือ
*   **Hardware Integration:** 
    *   ใช้ WebRTC (`navigator.mediaDevices.getUserMedia`) เพื่อดึงภาพจากกล้องมือถือมาสแกน QR และสลิป
    *   ใช้ HTML5 Geolocation API เพื่อดึงพิกัด (GPS) แบบ Real-time มาวิเคราะห์ Fake GPS
*   **State Management:** ใช้ **Zustand** หรือ React Context ในการจัดการสถานะ (เช่น สลับหน้าจอ Decoy, ยอดเงินคงเหลือจำลอง)

### 2. Backend (Core API Service)
*   **Framework:** **Node.js** ร่วมกับ **Express.js** ออกแบบเป็น RESTful API
*   **Security:** ใช้ JWT (JSON Web Tokens) จัดการ Session และใช้ไลบรารีเข้ารหัสเพื่อสร้าง **Micro-Cryptographic Watermark** (ลายน้ำดิจิทัล) แอบแฝงลงในรูปสลิปตอนที่กดโอนเงินสำเร็จ
*   **Core Logic Modules:**
    *   `Escrow Manager`: จัดการสถานะ Pending (หน่วงเวลา) ของธุรกรรม 15-30 นาที และ API สำหรับการดึงเงินกลับ (Recall)
    *   `Trust Engine`: คำนวณและดึงค่า Trust Score ของบัญชีปลายทาง
    *   `Duress Controller`: ตรวจจับการใช้รหัส PIN ฉุกเฉิน เพื่อสับสวิตช์ UI และยิง Webhook จำลองการแจ้งตำรวจ (Silent Alarm)

### 3. Database (Data Persistence)
*   **Database:** **MongoDB** (NoSQL) บริหารผ่าน Mongoose เพื่อความยืดหยุ่นในการเก็บข้อมูลที่โครงสร้างไม่ตายตัว เช่น Log การสแกนรูปภาพ, ประวัติธุรกรรม, และพิกัด GPS
*   **Deployment:** รันเป็น Container ผ่าน **Docker** เพื่อให้สามารถล้างข้อมูลและเซ็ตอัพสำหรับการทำเดโม่ได้อย่างรวดเร็ว

### 4. AI & Image Processing Service (Microservice บริหารโมเดล)
แยก Service ออกมาต่างหากเพื่อให้การรัน AI ไม่ไปดึงทรัพยากรการทำงานของระบบหลัก
*   **Framework:** **Python 3.10+** ร่วมกับ **FastAPI** เพื่อการตอบสนองที่รวดเร็วแบบ Asynchronous
*   **Computer Vision (OpenCV):**
    *   วิเคราะห์ Edge Detection หาขอบสติ๊กเกอร์ (QR-Overlay)
    *   วิเคราะห์ Histogram แสงสะท้อน (Screen Glare) เพื่อแยกระหว่างกระดาษกับหน้าจอมือถือ
    *   ใช้เทคนิค Error Level Analysis (ELA) วิเคราะห์พิกเซลภาพสลิปเพื่อหาการตัดต่อแต่งเลข
*   **Generative AI (Gemini 1.5 Flash):**
    *   ประมวลผลผ่าน SDK `google-genai` เพื่ออ่าน Text ในสลิป (OCR) และวิเคราะห์บริบทเปรียบเทียบกับฐานข้อมูลว่าสลิปนี้เมคขึ้นมาหรือไม่

### 5. Infrastructure & DevOps
*   **Containerization:** ห่อทุกระบบ (Frontend, Backend, AI Service, Database) ด้วย **Docker Compose** (`docker-compose.yml`) ทำให้สามารถรันระบบทั้งก้อนโชว์กรรมการได้เพียงพิมพ์คำสั่งเดียว (`docker-compose up`)
*   **CI/CD Pipeline:** เขียน Workflow ผ่าน **GitHub Actions** เพื่อจำลองกระบวนการ Automated Testing และ Build สะท้อนความเป็นมืออาชีพ
